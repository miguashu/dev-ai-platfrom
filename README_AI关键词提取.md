# AI智能关键词提取功能说明

## 🎯 功能概述

使用AI模型从用户自然语言查询中智能提取**核心检索关键词**，适用于文件检索、知识库问答、技术查询、故障排查等**所有场景**，大幅提升检索准确率。

## 🔧 技术实现

### 1. 核心组件

#### `AiKeywordExtractorService` - AI关键词提取服务
- **位置**: `src/main/java/com/devai/devaiplatform/service/AiKeywordExtractorService.java`
- **功能**: 调用ChatLanguageModel进行智能关键词提取
- **特点**: 
  - 通用化Prompt设计，支持所有查询类型
  - 自动清理AI响应中的多余字符
  - 异常时优雅降级
  - 提供 `extractKeyword()` 和 `extractFileNameKeyword()` 两个方法（向后兼容）

#### `HybridRetrievalService.extractFileNameKeyword()` - 混合提取策略
- **优先级1**: AI提取（更智能、更准确）
- **优先级2**: 规则匹配（兜底方案，确保可用性）

### 2. AI提示词设计（通用版）

```
你是一个智能关键词提取专家。请从用户的查询中提取最核心的检索关键词。

## 任务说明
用户会用自然语言提问或描述需求，你需要从中提取出最适合用于检索的核心关键词组合。
这些关键词将用于在知识库、文档库、代码库中进行精准检索。

## 提取原则
1. 保留核心实体：人名、项目名、技术名词、产品名、文档标题等
2. 保留关键动作/状态：配置、部署、报错、优化、异常等有检索价值的词
3. 去除无意义词汇：帮我、请、的、了、内容、信息、怎么、如何等
4. 保持语义完整性：提取的关键词组合应该能独立表达查询意图
5. 简洁精炼：通常2-10个词
6. 不要添加任何解释、标点或额外文字

## 多场景示例

输出：李腾涛简历

输出：数据库连接超时
```

## 🔄 工作流程

```
用户查询: "帮我分析一下李腾涛简历的内容"
    ↓
[Step 1] 尝试AI提取
    ↓ 调用ChatLanguageModel
AI响应: "李腾涛简历"
    ↓
[Step 2] 验证AI结果
    ✓ 长度合理 (2-50字符)
    ✓ 非空非null
    ↓
[Step 3] 使用AI结果进行wildcard匹配
    query: "李腾涛简历*"
    ↓
[Step 4] 返回精确匹配的文件片段
    ✅ 只匹配"李腾涛简历.pdf"
    ❌ 不匹配"李腾涛个人简历.pdf"
```

## 🛡️ 容错机制

### AI失败时的降级策略

```java
// 优先级1: AI提取
if (aiKeywordExtractorService != null) {
    String aiKeyword = aiKeywordExtractorService.extractFileNameKeyword(query);
    if (aiKeyword != null) return aiKeyword;
}

// 优先级2: 规则匹配（兜底）
return extractFileNameKeywordByRule(query);
```

**降级触发条件**:
- AI服务不可用（Bean未注入）
- AI调用超时或异常
- AI返回null或无效结果
- 网络问题导致API调用失败

## 📝 配置说明

### 可选优化

如需调整AI行为，可修改 `AiKeywordExtractorService.buildExtractionPrompt()` 中的提示词。

## 🚀 性能影响

- **AI提取耗时**: 约200-500ms（取决于网络和模型）
- **规则匹配耗时**: <1ms
- **总体影响**: 首次查询增加约0.3s，但大幅提升准确率，减少误召回
