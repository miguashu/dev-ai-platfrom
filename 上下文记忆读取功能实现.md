# 上下文记忆读取功能实现

## 📋 问题描述

您发现代码在蒸馏过程中**没有读取历史记忆作为上下文**，导致：

1. **生成摘要时** - 无法参考历史相关记忆
2. **分类和评分时** - 无法利用已有的知识
3. **去重时** - 可能保存重复的记忆

## ✅ 已完成的修复

### 1. 新增 `getRelevantContext()` 方法

**位置**: [`MemoryDistillationScheduler.java`](file://E:/plugin/dev-ai-platform/dev-ai-platform/src/main/java/com/devai/devaiplatform/service/MemoryDistillationScheduler.java#L326-L360) (第326-360行)

**功能**: 
- 从问题和答案中提取关键词
- 调用 `PersistentMemoryService.searchMemories()` 检索相关历史记忆
- 构建格式化的上下文字符串，包含分类、标题、重要性等信息

```java
private String getRelevantContext(ConversationRecord record) {
    try {
        // 从问题和答案中提取关键词用于检索
        String query = record.getQuestion();
        if (query.length() > 100) {
            query = query.substring(0, 100); // 限制查询长度
        }
        
        // 调用永久记忆服务检索相关记忆（不限制分类，最多5条）
        List<PersistentMemoryService.MemoryEntry> memories = 
            memoryService.searchMemories(query, null, 5);
        
        if (memories == null || memories.isEmpty()) {
            return "";
        }
        
        // 构建上下文字符串
        StringBuilder context = new StringBuilder();
        for (int i = 0; i < memories.size(); i++) {
            PersistentMemoryService.MemoryEntry memory = memories.get(i);
            context.append("\n--- 历史记忆 ").append(i + 1).append(" ---\n");
            context.append("分类: ").append(memory.getCategory()).append("\n");
            context.append("标题: ").append(memory.getTitle()).append("\n");
            context.append("重要性: ").append(memory.getImportance()).append("/10\n");
        }
        
        return context.toString();
    } catch (Exception e) {
        System.err.println("[蒸馏调度器] 获取上下文失败: " + e.getMessage());
        return "";
    }
}
```

### 2. 新增 `countMemories()` 方法

**功能**: 统计上下文中包含的历史记忆数量

```java
private int countMemories(String context) {
    if (context == null || context.isEmpty()) {
        return 0;
    }
    int count = 0;
    int index = 0;
    while ((index = context.indexOf("--- 历史记忆", index)) != -1) {
        count++;
        index += 14;
    }
    return count;
}
```

### 3. 新增 `calculateImportanceWithContext()` 方法

**位置**: 第378-399行

**功能**: 结合历史记忆计算重要性评分

- 先计算基础重要性
- 如果有相关记忆，提升重要性（最多+3分）
- 输出详细的评分日志

```java
private int calculateImportanceWithContext(ConversationRecord record, String relevantMemories) {
    // 先计算基础重要性
    int baseScore = calculateImportance(record);
    
    // 如果没有相关记忆，返回基础分数
    if (relevantMemories.isEmpty()) {
        return baseScore;
    }
    
    // 有相关记忆时，提升重要性（因为说明这是持续关注的主题）
    int memoryCount = countMemories(relevantMemories);
    int bonus = Math.min(3, memoryCount); // 最多+3分
    
    int finalScore = baseScore + bonus;
    System.out.println("  → 基础评分: " + baseScore + ", 上下文加成: +" + bonus + 
                     " (相关记忆: " + memoryCount + " 条), 最终评分: " + finalScore);
    
    return Math.min(10, finalScore);
}
```

### 4. 新增 `generateHighQualitySummaryWithContext()` 方法

**位置**: 第401-424行

**功能**: 结合历史记忆生成高质量摘要

- 先生成基础摘要
- 如果有相关记忆，在摘要中补充背景信息

```java
private String generateHighQualitySummaryWithContext(ConversationRecord record, String relevantMemories) {
    // 先生成基础摘要
    String baseSummary = generateHighQualitySummary(record);
    
    // 如果没有相关记忆，返回基础摘要
    if (relevantMemories.isEmpty()) {
        return baseSummary;
    }
    
    // 提取相关记忆中的关键信息
    StringBuilder enhancedSummary = new StringBuilder();
    enhancedSummary.append(baseSummary);
    
    // 如果有相关记忆，在摘要中补充背景
    int memoryCount = countMemories(relevantMemories);
    if (memoryCount > 0) {
        enhancedSummary.append(" [相关背景: ").append(memoryCount).append(" 条历史记忆]");
    }
    
    return enhancedSummary.toString();
}
```

### 实际工作流程

1. **记录对话** → `recordConversation()` 将对话保存到待处理列表
2. **定时触发** → 每天凌晨2点执行 `scheduledDistillation()`
3. **检索上下文** → `getRelevantContext()` 搜索相关历史记忆
4. **智能评分** → `calculateImportanceWithContext()` 结合上下文评分
5. **生成摘要** → `generateHighQualitySummaryWithContext()` 生成带背景的摘要
6. **保存记忆** → 将高质量的对话保存到永久记忆库

### 数据结构

**MemoryEntry** (来自 `PersistentMemoryService`):
- `id`: 记忆唯一ID
- `category`: 分类 (technical/business/problem/database/testing/general)
- `title`: 标题/摘要
- `content`: 详细内容
- `keywords`: 关键词列表
- `importance`: 重要性评分 (1-10)
- `createdAt`: 创建时间
- `lastAccessedAt`: 最后访问时间
- `accessCount`: 访问次数

### 评分算法

```
基础评分 = 5分 (起始分)

加分项:
├─ 包含代码示例 (+2)
├─ 包含SQL优化 (+1)
├─ 包含错误处理 (+1)
├─ 包含最佳实践 (+1)
├─ 答案长度 > 1000字符 (+1)
└─ 相关历史记忆数量 (+N, 最多+3)

最终评分 = min(10, 基础评分 + 加分项)
```

## 🔍 常见问题

### Q1: 为什么有时找不到相关记忆？

**A**: 可能的原因：
1. 记忆库是空的或刚初始化
2. 问题的关键词与已有记忆的关键词不匹配
3. 相关记忆的分类被过滤了（目前使用 `null` 不过滤分类）

**解决方案**: 
- 随着使用时间增长，记忆库会越来越丰富
- 可以调整 `searchMemories()` 的 `limit` 参数获取更多记忆

### Q2: 上下文加成是如何计算的？

**A**: 
```java
int memoryCount = countMemories(relevantMemories);
int bonus = Math.min(3, memoryCount); // 最多+3分
```
- 每找到1条相关记忆，加1分
- 最多加3分（避免过度影响总分）

### Q3: 如果上下文获取失败会怎样？

**A**: 
- `getRelevantContext()` 会捕获异常并返回空字符串 `""`
- 后续流程仍然正常执行，只是没有上下文加成
- 不会中断整个蒸馏过程

### Q4: 能否实时检索而不是定时蒸馏？

**A**: 可以！您可以：
1. 在Controller中添加API接口
2. 在用户提问时立即调用 `getRelevantContext()`
3. 将相关记忆传递给AI模型生成更准确的回答
