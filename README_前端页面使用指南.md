# 前端页面使用指南

## 📋 功能概述

前端页面提供了完整的RAG系统交互界面，包含以下功能：

1. **实时对话** - 类似ChatGPT的聊天界面
2. **单文件上传** - 上传单个PDF到知识库
3. **批量OCR处理** - 批量处理文件夹中的图片PDF
4. **上下文记忆** - 保持对话历史
5. **状态监控** - 实时显示系统状态

---

## 🚀 启动方式

### 1. 启动后端服务

```bash
cd E:/plugin/dev-ai-platform/dev-ai-platform
mvn spring-boot:run
```


## 💬 核心功能使用

### 1️⃣ 实时对话（主功能）

**位置**：页面中央聊天区域

**使用方法**：
1. 在底部输入框输入问题
2. 按 `Enter` 或点击"发送"按钮
3. 等待AI回复

**支持的提问类型**：
- 代码查询："查找用户管理模块的代码"
- SQL优化："如何优化这个MySQL查询？"
- 错误排查："java.lang.NullPointerException怎么解决？"
- 需求分析："帮我设计一个订单表结构"
- 代码生成："生成User表的CRUD代码"

**特点**：
- ✅ 自动保存对话历史
- ✅ 支持Markdown格式渲染
- ✅ 显示消息时间戳
- ✅ 对话内容自动记录用于蒸馏

---

### 2️⃣ 单文件上传

**位置**：左侧边栏 → "📄 上传PDF到知识库"

**使用方法**：
1. 点击"📤 选择并上传PDF"按钮
2. 选择要上传的PDF文件
3. 等待上传和解析完成

**处理流程**：
```
选择PDF文件
    ↓
上传到服务器临时目录 (./uploads/temp)
    ↓
智能文本提取（自动判断是否需要OCR）
    ↓
数据清洗（去除噪声）
    ↓
三级分片（L1/L2/L3）
    ↓
向量化并存入知识库
    ↓
上传成功提示
```

**预期效果**：

**成功时：**
- 进度条显示 100%
- 日志显示：`✅ PDF上传并入库成功: xxx.pdf`
- AI自动回复确认消息

**失败时：**
- 日志显示具体错误原因
- AI提示失败信息

**支持的文件**：
- 格式：仅支持 `.pdf`
- 大小：无限制（建议 < 50MB）
- 类型：文本PDF、扫描版PDF、图片PDF均可

---

### 3️⃣ 批量OCR处理

**位置**：左侧边栏 → "📁 批量OCR处理"

**使用方法**：
1. 点击"📂 选择文件夹批量处理"按钮
2. 选择包含多个PDF的文件夹
3. 等待批量处理完成

**处理流程**：
```
选择文件夹
    ↓
检测所有PDF文件
    ↓
逐个进行OCR识别
    ↓
智能判断是否需要OCR
    ↓
批量入库
    ↓
显示详细结果列表
```

**预期输出**：
```
[14:30:25] 检测到 15 个PDF文件
[14:30:25] 开始批量OCR处理...
[14:30:30] ✅ 处理完成: 15 个文件
[14:30:30] [1] document1.pdf: success
[14:30:30] [2] document2.pdf: success
[14:30:30] [3] scan_document.pdf: success (OCR)
...
```

**适用场景**：
- 大量扫描版PDF需要识别
- 图片型合同、文档批量处理
- 纸质文档数字化

---

### 4️⃣ 状态监控面板

**位置**：左侧边栏底部 → "📊 系统状态"

**显示内容**：

| 指标 | 说明 | 刷新频率 |
|------|------|---------|
| **待蒸馏对话** | 等待批量蒸馏的对话数量 | 10秒 |
| **永久记忆** | 已保存的永久记忆数量 | 10秒 |
| **对话轮数** | 当前会话的对话次数 | 实时 |

**示例**：
```

```

## 🔧 配置说明

### 后端配置

**application.properties** 中无需特殊配置，默认即可：

```properties
server.port=8081

# Ollama配置
langchain4j.ollama.base-url=http://127.0.0.1:11434
langchain4j.ollama.chat-model.name=qwen2.5:7b
langchain4j.ollama.embedding-model.name=nomic-embed-text
```

### 文件上传限制（可选）

如需限制上传文件大小，在 `application.properties` 添加：

```properties
# 单个文件最大50MB
spring.servlet.multipart.max-file-size=50MB

# 总请求最大100MB
spring.servlet.multipart.max-request-size=100MB
```

## 🐛 常见问题

### Q1: 文件上传后找不到

**原因**：文件保存到临时目录，重启后会丢失

**解决**：
- 重要文件请备份到 `./docs` 文件夹
- 或使用"批量OCR处理"从固定目录读取

---

### Q2: 对话没有上下文

**原因**：每次对话是独立的，需要手动维护上下文

**解决**：
- 在提问时引用之前的内容
- 例如："刚才说的用户表，再加一个字段"

---

### Q3: PDF上传后检索不到内容

**可能原因**：
1. PDF是纯图片（未启用OCR）
2. PDF加密了
3. PDF内容为空

**解决方法**：
- 检查上传日志中的文本提取方式
- 如果是图片PDF，使用"批量OCR处理"
- 手动测试PDF是否可以复制文字

---

### Q4: 页面打不开

**检查步骤**：
1. 确认后端服务已启动
2. 检查端口8081是否被占用
3. 浏览器控制台查看网络错误

**解决方法**：
```bash
# 检查端口
netstat -ano | findstr :8081

# 重启服务
mvn spring-boot:run
```

## 📈 性能优化建议

### 1. 大批量文件处理

**建议**：
- 单次不超过50个文件
- 每个文件 < 50MB
- 分批处理，每批间隔10秒

### 2. 对话历史管理

**建议**：
- 每轮对话控制在500字以内
- 重要内容使用"记住"关键词触发蒸馏
- 定期清理浏览器缓存

### 3. 内存优化

**监控指标**：
- 待蒸馏对话 < 100条
- 永久记忆 < 2000条
- JVM堆内存使用 < 80%

前端添加按钮：
```html
<button onclick="exportChat()">导出对话</button>
```

实现JavaScript：
```javascript
function exportChat() {
    const data = JSON.stringify(chatHistory, null, 2);
    const blob = new Blob([data], {type: 'application/json'});
    const url = URL.createObjectURL(blob);
    const a = document.createElement('a');
    a.href = url;
    a.download = 'chat_history.json';
    a.click();
}
```
