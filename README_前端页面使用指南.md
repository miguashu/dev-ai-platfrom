# 前端页面使用指南

## 📋 功能概述

项目提供两个前端页面，覆盖完整的 AI 交互与管理体验：

1. **智能对话页面（index.html）** — 类似 ChatGPT 的聊天界面，支持 QA/Agent 双模式
2. **Agent 管理面板（agent-dashboard.html）** — Agent 配置可视化、自愈规则、升级策略展示

### index.html 功能

1. **实时对话** - 类似ChatGPT的聊天界面，支持 QA 模式 / Agent 模式切换
2. **联网搜索** - 按需获取最新网络信息
3. **单文件上传** - 上传单个PDF到知识库
4. **批量OCR处理** - 批量处理文件夹中的图片PDF
5. **上下文记忆** - 保持对话历史，历史会话管理
6. **状态监控** - 实时显示系统状态

### agent-dashboard.html 功能

1. **Agent 元信息** - Agent 名称、版本、描述
2. **感知层** - 4 个传感器配置（OCR、日志、系统监控）
3. **认知层** - 规划引擎 + ES 知识库连接状态
4. **执行层** - 5 个执行工具清单
5. **自愈机制** - 健康阈值进度条、4 步诊断流程、4 项修复策略
6. **自升级机制** - 升级策略、触发器、7 步流程
7. **通信协议** - API Key 认证信息

## 💬 核心功能使用

### 1️⃣ 实时对话（主功能）

**位置**：页面中央聊天区域

**使用方法**：
1. 在底部输入框输入问题
2. 按 `Enter` 或点击"发送"按钮
3. 等待AI回复

**模式切换**：
- **💬 智能问答模式** — 直接对话，快速回答，支持联网搜索
- **🛠️ Agent 模式** — 智能体调度，可执行任务、调用工具

**功能开关**：
- **知识库检索** — 开启后自动检索 RAG 向量库相关文档
- **🌐 联网搜索** — 开启后获取最新网络信息（优先级高于 RAG）

**支持的提问类型**：
- 代码查询："查找用户管理模块的代码"
- SQL优化："如何优化这个MySQL查询？"
- 错误排查："java.lang.NullPointerException怎么解决？"
- 需求分析："帮我设计一个订单表结构"
- 代码生成："生成User表的CRUD代码"

**特点**：
- ✅ 自动保存对话历史
- ✅ 支持Markdown格式渲染
- ✅ 显示消息时间戳
- ✅ 对话内容自动记录用于蒸馏

---

### 2️⃣ 单文件上传

**位置**：左侧边栏 → "📄 上传PDF到知识库"

**使用方法**：
1. 点击"📤 选择并上传PDF"按钮
2. 选择要上传的PDF文件
3. 等待上传和解析完成

**处理流程**：
```
选择PDF文件
    ↓
上传到服务器临时目录 (./uploads/temp)
    ↓
智能文本提取（自动判断是否需要OCR）
    ↓
数据清洗（去除噪声）
    ↓
三级分片（L1/L2/L3）
    ↓
向量化并存入知识库
    ↓
上传成功提示
```

**预期效果**：

**成功时：**
- 进度条显示 100%
- 日志显示：`✅ PDF上传并入库成功: xxx.pdf`
- AI自动回复确认消息

**失败时：**
- 日志显示具体错误原因
- AI提示失败信息

**支持的文件**：
- 格式：仅支持 `.pdf`
- 大小：无限制（建议 < 50MB）
- 类型：文本PDF、扫描版PDF、图片PDF均可

---

### 3️⃣ 批量OCR处理

**位置**：左侧边栏 → "📁 批量OCR处理"

**使用方法**：
1. 点击"📂 选择文件夹批量处理"按钮
2. 选择包含多个PDF的文件夹
3. 等待批量处理完成

**处理流程**：
```
选择文件夹
    ↓
检测所有PDF文件
    ↓
逐个进行OCR识别
    ↓
智能判断是否需要OCR
    ↓
批量入库
    ↓
显示详细结果列表
```

**预期输出**：
```
[14:30:25] 检测到 15 个PDF文件
[14:30:25] 开始批量OCR处理...
[14:30:30] ✅ 处理完成: 15 个文件
[14:30:30] [1] document1.pdf: success
[14:30:30] [2] document2.pdf: success
[14:30:30] [3] scan_document.pdf: success (OCR)
...
```

**适用场景**：
- 大量扫描版PDF需要识别
- 图片型合同、文档批量处理
- 纸质文档数字化

## 🔧 配置说明

### 后端配置

**application.properties** 中无需特殊配置，默认即可：

```properties
server.port=8081

# Ollama配置
langchain4j.ollama.base-url=http://127.0.0.1:11434
langchain4j.ollama.chat-model.name=qwen2.5:7b
langchain4j.ollama.embedding-model.name=nomic-embed-text
```

### 文件上传限制（可选）

如需限制上传文件大小，在 `application.properties` 添加：

```properties
# 单个文件最大50MB
spring.servlet.multipart.max-file-size=50MB

# 总请求最大100MB
spring.servlet.multipart.max-request-size=100MB
```

## 🐛 常见问题

### Q1: 文件上传后找不到

**原因**：文件保存到临时目录，重启后会丢失

**解决**：
- 重要文件请备份到 `./docs` 文件夹
- 或使用"批量OCR处理"从固定目录读取

---

### Q2: 对话没有上下文

**原因**：每次对话是独立的，需要手动维护上下文

**解决**：
- 在提问时引用之前的内容
- 例如："刚才说的用户表，再加一个字段"

---

### Q3: PDF上传后检索不到内容

**可能原因**：
1. PDF是纯图片（未启用OCR）
2. PDF加密了
3. PDF内容为空

**解决方法**：
- 检查上传日志中的文本提取方式
- 如果是图片PDF，使用"批量OCR处理"
- 手动测试PDF是否可以复制文字

---

### Q4: 页面打不开

**检查步骤**：
1. 确认后端服务已启动
2. 检查端口8081是否被占用
3. 浏览器控制台查看网络错误

**解决方法**：
```bash
# 检查端口
netstat -ano | findstr :8081

# 重启服务
mvn spring-boot:run
```

## 📈 性能优化建议

### 1. 大批量文件处理

**建议**：
- 单次不超过50个文件
- 每个文件 < 50MB
- 分批处理，每批间隔10秒

### 2. 对话历史管理

**建议**：
- 每轮对话控制在500字以内
- 重要内容使用"记住"关键词触发蒸馏
- 定期清理浏览器缓存

### 3. 内存优化

**监控指标**：
- 待蒸馏对话 < 100条
- 永久记忆 < 2000条
- JVM堆内存使用 < 80%

### 2. 添加新功能

**示例：添加导出对话功能**

前端添加按钮：
```html
<button onclick="exportChat()">导出对话</button>
```

实现JavaScript：
```javascript
function exportChat() {
    const data = JSON.stringify(chatHistory, null, 2);
    const blob = new Blob([data], {type: 'application/json'});
    const url = URL.createObjectURL(blob);
    const a = document.createElement('a');
    a.href = url;
    a.download = 'chat_history.json';
    a.click();
}
```

## 🖥️ Agent 可视化管理面板

访问 `http://localhost:8081/agent-dashboard.html` 查看 Agent 管理面板。

### 页面布局

- **顶部状态栏** — Agent 状态、自愈开关、升级开关
- **Agent 元信息卡片** — 名称、版本、描述、创建时间
- **感知层卡片** — 4 个传感器配置（用户输入、日志分析、OCR、系统监控）
- **认知层卡片** — 规划引擎（树状思维链）、ES 向量知识库连接
- **执行层卡片** — 5 个执行工具清单（文件操作、Shell、OCR、ES管理、缓存清理）
- **自愈机制卡片** — 健康阈值进度条、4 步故障诊断、4 项修复策略
- **自升级机制卡片** — 升级策略（manual/auto）、触发器、7 步流程
- **通信协议卡片** — API 端点、认证方式

### 数据来源

面板数据**动态加载**自后端 API：
```
GET /api/dev-ai/agent/config          → 完整配置 JSON
GET /api/dev-ai/agent/config/summary   → 配置摘要
```

修改 `agent-config.json` 并重启服务，面板自动反映最新配置。

### 配置修改示例

| 操作 | 修改文件 | 修改项 |
|------|---------|--------|
| 禁用自升级 | `agent-config.json` | `self_upgrade_mechanism.enabled` → `false` |
| 调整内存阈值 | `agent-config.json` | `thresholds.memory_usage_percent` |
| 切换升级策略 | `agent-config.json` | `upgrade_policy.type` → `"manual"` |
