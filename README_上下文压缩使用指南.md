# 上下文压缩功能使用指南

## 📋 功能简介

上下文压缩功能允许用户手动触发对话上下文的智能压缩，当对话过长时提取关键信息并生成结构化摘要，保留有价值的技术内容。

## 🎯 使用场景

1. **长对话优化**：当对话超过20条消息或15000字符时，压缩上下文释放空间
2. **话题切换前**：在切换到新话题前，压缩之前的对话保留精华
3. **会话恢复**：重新打开历史会话时，快速恢复上下文

## 🖥️ 前端操作步骤

### 方式一：检查压缩状态

1. 在左侧边栏找到 **"🗜️ 上下文管理"** 面板
2. 点击 **"🔍 检查压缩状态"** 按钮
3. 系统会显示当前会话的统计信息：
   - 📊 消息数量 / 阈值（20条）
   - 📏 总字符数 / 阈值（15000字符）
   - ⚠️ 是否需要压缩的建议

### 方式二：手动压缩上下文

1. 确保当前有活动的会话（新建对话或选择历史会话）
2. 点击 **"⚡ 手动压缩上下文"** 按钮
3. 确认压缩操作
4. 等待AI处理（可能需要几秒钟）
5. 查看压缩结果：
   - ✅ 压缩完成提示
   - 📦 原始消息数量
   - 🗜️ 压缩率
   - 📋 压缩摘要预览

## 🔌 API接口说明

### 1. 检查是否需要压缩
```bash
GET /api/dev-ai/context/should-compress?sessionId={sessionId}
```

**返回示例：**
```json
{
  "code": 200,
  "data": {
    "sessionId": "abc123",
    "shouldCompress": true,
    "messageCount": 25,
    "totalChars": 18500,
    "thresholdMessages": 20,
    "thresholdChars": 15000
  }
}
```

### 2. 手动触发压缩
```bash
POST /api/dev-ai/context/compress?sessionId={sessionId}
```

**返回示例：**
```json
{
  "code": 200,
  "data": {
    "sessionId": "abc123",
    "originalMessageCount": 25,
    "compressedSummary": "### 📋 对话摘要\n...",
    "compressionStats": {
      "compressionRatio": "96.0%",
      "originalMessageCount": 25,
      "compressedMessageCount": 1
    }
  }
}
```

### 3. 获取压缩统计
```bash
GET /api/dev-ai/context/stats?sessionId={sessionId}
```

### 4. 清除压缩状态
```bash
DELETE /api/dev-ai/context/state?sessionId={sessionId}
```

## 💡 最佳实践

### 何时使用压缩？

✅ **推荐场景：**
- 对话超过20条消息
- 准备切换到完全不同的话题
- 感觉AI响应变慢或遗忘之前的内容
- 想要保存当前对话的精华

❌ **不推荐场景：**
- 对话刚开始（少于10条消息）
- 正在讨论同一个问题的细节
- 需要保留完整的对话记录用于审计

### 压缩后的注意事项

1. **摘要不是完整记录**：压缩会丢失一些细节，只保留关键信息
2. **可以继续对话**：压缩后可以继续在同一会话中对话
3. **多次压缩**：可以多次压缩，每次都会基于当前上下文生成新摘要
4. **结合记忆蒸馏**：重要内容建议同时使用"数据蒸馏"保存到永久记忆

## 🐛 常见问题

### Q1: 压缩失败怎么办？
**A:** 检查以下几点：
- 确保当前有活动的会话
- 检查网络连接是否正常
- 查看控制台日志是否有错误信息
- AI服务是否正常运行

### Q2: 压缩后信息丢失了？
**A:** 压缩是有损的，会丢失一些细节。如果某些信息很重要：
- 在压缩前先复制保存
- 使用"数据蒸馏"功能将重要内容保存到永久记忆
- 调整Prompt模板，增加关键信息识别规则

### Q3: 可以撤销压缩吗？
**A:** 目前不支持撤销。但你可以：
- 从历史会话中重新加载未压缩的版本
- 清除压缩状态后重新对话

### Q4: 压缩会影响后续对话吗？
**A:** 不会负面影响。压缩后的摘要会作为上下文的一部分，帮助AI理解历史背景。
